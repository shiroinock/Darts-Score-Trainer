## 評価タスク

このサブエージェントの実行結果を評価し、`.claude/agents/local-ci-checker.md` の指示内容に改善すべき点があれば、そのファイルに追記してください。

## 評価観点

### 1. サブエージェントの並列起動
- 3つのサブエージェント（biome-check、test-check、build-check）が並列起動されているか
- 単一のメッセージで3つのTaskツール呼び出しが行われているか
- 全てのサブエージェントの完了を待ってから結果を報告しているか

### 2. local-ci スキルの参照
- 最初に local-ci スキルを参照しているか
- スキルの最新の指示に従っているか

### 3. 結果の集計と報告
- 各サブエージェントの結果を正確に集計しているか
- 全てのチェック結果をまとめて報告しているか
- 失敗したチェックの詳細が明確に報告されているか

### 4. エラーメッセージの有用性
- Biome check失敗時に `npm run check:fix` の提案があるか
- Test失敗時に具体的なテストエラーが報告されているか
- Build失敗時に型エラー・ビルドエラーの詳細があるか
- 複数のチェックが失敗した場合、全ての問題がまとめて報告されているか

### 5. サマリー表示
- 成功時のサマリーが適切に表示されているか
- 失敗時のサマリーに全てのチェック結果が含まれているか
- 次に何をすべきかが明確か（失敗した場合）

## 改善アクション

評価結果に基づき、以下のいずれかを実行してください：

### 問題がある場合
`.claude/agents/local-ci-checker.md` に以下を追記：

1. **サブエージェント並列起動に関する改善**
   - 例: 「3つのTaskツール呼び出しを単一のメッセージで実行してください」
   - 例: 「全てのサブエージェントの完了を待ってから結果を集計してください」
   - 例: 「biome-check、test-check、build-checkを必ず使用してください」

2. **結果集計の改善**
   - 例: 「各サブエージェントの結果を正確に集計してください」
   - 例: 「複数のサブエージェントが失敗した場合、全ての問題をまとめて表示してください」

3. **サマリー表示の改善**
   - 例: 「成功時のサマリーには各チェックの結果を含めてください」
   - 例: 「失敗時のサマリーには修正方法を含めてください」

4. **local-ci スキル参照の改善**
   - 例: 「最初に必ず local-ci スキルを参照してください」
   - 例: 「スキルの指示に従った実装手順を厳密に守ってください」

### 問題がない場合
- 何もしないでください

## 重要な注意事項

- サブエージェント(Task tool)を再帰的に呼び出さないでください
- Edit ツールと Read ツールのみを使用してください
- ファイルの変更は `.claude/agents/local-ci-checker.md` のみに限定してください
